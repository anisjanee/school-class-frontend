# School Class Journal (Frontend)

Веб‑приложение “Журнал класса” для учителя: ученики, предметы, оценки/посещаемость, аналитика и календарь. Работает как статический сайт (без сервера).

## Возможности

- **Ученики**: добавление/удаление, имена на **2 языках** (тоҷикӣ/русский).
- **Предметы**: добавление/удаление, теги предметов.
- **Расписание**: слоты времени + привязка предметов по дням недели (Пн–Сб).
- **Оценки**: таблица оценок 1–10, итоговая средняя, быстрый ввод с клавиатуры.
- **Посещаемость**: режим отметок присутствия.
- **Фильтры**: выбор предмета, четверть/год + произвольный диапазон дат, поиск ученика.
- **Графики/аналитика**: график оценок, рейтинг, сравнение двух учеников.
- **Календарь**: учебные дни/праздники/без уроков, навигация по месяцам.
- **Автосохранение** и кнопки “Сохранить”.
- **Экспорт/импорт JSON** (резервная копия).
- **Тема**: светлая/тёмная (с сохранением выбора).

## Технологии

- HTML/CSS/JavaScript (без сборки)
- [Chart.js](https://www.chartjs.org/) для графиков
- `localStorage` для хранения данных у пользователя

## Как запустить локально

### Вариант 1 (проще всего)

Открой файл `index.html` двойным кликом в браузере.

### Вариант 2 (рекомендуется — через локальный сервер)

Чтобы всё работало стабильно (особенно импорт/экспорт в некоторых браузерах), подними простой сервер:

```bash
python -m http.server 3000
```

Открой в браузере `http://localhost:3000`.

## Данные и сохранение

- Данные сохраняются **локально в браузере** (через `localStorage`).
- Это значит, что у каждого пользователя будет **свой** журнал на своём устройстве/браузере.
- Для переноса данных между устройствами используй **Экспорт JSON** → на другом устройстве **Импорт JSON**.

## Структура проекта

- `index.html` — разметка страниц (setup/grades/analytics/calendar)
- `style.css` — стили (включая тёмную тему)
- `script.js` — вся логика приложения
